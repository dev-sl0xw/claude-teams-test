# 소크라테스식 학습 노트 작성 가이드

## 소크라테스식 방법이란?

> "나는 아무것도 가르치지 않는다. 나는 다만 그들이 스스로 생각하게 할 뿐이다." - 소크라테스

이 프로젝트의 모든 학습 노트와 코드 주석은 **소크라테스식 질문법**을 사용합니다.
단순히 "이것을 이렇게 하세요"가 아니라, **"왜 이것을 이렇게 해야 하는가?"**를 끊임없이 묻습니다.

## 왜 소크라테스식 방법인가?

AWS SAA 시험은 단순 암기가 아니라 **시나리오 기반의 의사결정 능력**을 평가합니다.
"S3의 스토리지 클래스 종류를 나열하시오"가 아니라,
**"주로 90일 이후에 접근하는 로그 데이터에 가장 비용 효율적인 스토리지 클래스는?"** 같은 문제가 출제됩니다.

따라서 "무엇(What)"보다 **"왜(Why)"와 "언제(When)"**를 이해하는 것이 중요합니다.

## 노트 작성 템플릿

모든 학습 노트는 다음 구조를 따릅니다:

```markdown
# 주제 제목

## 왜 이것이 중요한가? (질문)
> 읽기 전에 스스로에게 물어보세요: [도발적 질문]
>
> 예시: "만약 서버가 1대뿐이고, 그 서버가 있는 데이터센터에 화재가 발생하면 어떻게 되나요?"

## 핵심 개념
[왜(Why) 중심으로 설명합니다]
- 이 기술/서비스가 해결하는 문제
- 이것이 없었다면 어떤 일이 벌어지는지
- 다른 대안과 비교했을 때의 장단점

## 잠깐, 그런데... (흔한 오해)
[SAA 시험에서 자주 혼동되는 포인트]
- ❌ 잘못된 이해: ...
- ✅ 올바른 이해: ...

## 핸즈온 연결
> 이 개념은 [Lab XX](../labs/labXX-이름/README.md)에서 구현됩니다.
> 실습 전에 예측해보세요:
> - 이 패턴을 적용하지 않으면 어떤 장애 시나리오가 발생할까?
> - 이 설정을 변경하면 비용에 어떤 영향이 있을까?

## SAA 시험 핵심 포인트
- [ ] 시험에 나올 수 있는 핵심 지식 1
- [ ] 시험에 나올 수 있는 핵심 지식 2
- [ ] 시험에 나올 수 있는 핵심 지식 3

## 관련 Pillar
- [보안 Pillar](../../02-security/notes/관련-노트.md) - 이 개념이 보안에 미치는 영향
- [비용 최적화 Pillar](../../05-cost-optimization/notes/관련-노트.md) - 비용 트레이드오프
```

## 코드 주석 작성 가이드

모든 CDK 코드의 주석은 다음 패턴을 따릅니다:

### 1. "왜(Why)" 주석

```typescript
// -- 왜: [이 설정이 존재하는 이유를 한 문장으로]
// [구체적인 설명: 이것이 없으면 어떤 문제가 발생하는지]
// 질문: [학습자가 스스로 생각해볼 질문]
```

### 2. SAA 포인트 주석

```typescript
// SAA 포인트: [시험에서 이 개념이 어떻게 출제되는지]
```

### 3. Pillar 연결 주석

```typescript
// [보안 Pillar] 이 설정은 데이터 보호 패턴의 일부입니다.
// [비용 최적화 Pillar] 이 옵션을 활성화하면 추가 비용이 발생합니다.
```
